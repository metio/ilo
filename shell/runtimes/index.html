Runtimes

Talks about: docker, lxd, podman, and runtime

ilo shell by default searches your local system for supported runtimes. It prefers podman over docker over lxd at the moment. In order to force the usage of a specific runtime, use the --runtime flag.

Docker

Force ilo to use docker like this:

$ ilo shell --runtime docker

# alias
$ ilo shell --runtime d

Important: In case your image uses root as its user, and you cannot run rootless docker (introduced with 19.03), use the --run-as option to override the user in the image.

Podman

Force ilo to use podman like this:

$ ilo shell --runtime podman

# alias
$ ilo shell --runtime p

Important: In case your images uses root as its user, and you cannot run rootless podman, use the --run-as option to override the user in the image. Take a look at this article for an in-depth example of how rootless work with podman.
